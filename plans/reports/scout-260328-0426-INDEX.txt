FRAME PROCESSORS (PLUGGABLE PIPELINE)
=====================================

face_swapper (ALWAYS ENABLED):
  - Detects faces in target frame
  - Swaps with source face embedding
  - Applies Poisson blending
  - Optional color correction
  - Thread-local frame interpolation for smoothing

face_enhancer (OPTIONAL):
  - Upscales and denoises swapped face
  - Uses GFPGAN ONNX model
  - Rate-limited with semaphores on CPU

face_enhancer_gpen256, face_enhancer_gpen512 (OPTIONAL):
  - Alternative enhancement variants
  - GPEN models instead of GFPGAN

face_masking (OPTIONAL):
  - Preserves original mouth over swapped face
  - Detects mouth region from facial landmarks
  - Feathering for smooth blending
  - Complex facial geometry handling

Processing strategy:
  - Processors form a pipeline (user-selectable)
  - Each processor implements: pre_check(), pre_start(), process_frame()
  - Dynamic module loading via importlib
  - ThreadPoolExecutor for batch frame processing

THREADING & CONCURRENCY
========================

Live Mode (Real-time):
  Sequential pipeline (no parallelization):
    VideoCapturer.read() -> face_analyser -> face_swapper -> 
    face_enhancer (optional) -> gpu_processing -> virtual_camera.send()
  Runs in main GUI thread (single-threaded)

Batch Mode (Video/Image):
  1. Extract frames: main thread
  2. ThreadPoolExecutor (batch_size = min(32, N_frames/workers))
  3. Process frame: worker threads
  4. Re-encode: ffmpeg (main thread)

Synchronization:
  - MAP_LOCK: RwLock protecting source_target_map mutations
  - THREAD_LOCK: Face swapper model initialization
  - FACE_ANALYSER_LOCK: Face detection model initialization
  - Thread-local storage: Frame interpolation state (per-thread previous frame)

Batch Processing Detail:
  - Batch size auto-calculated: max(1, min(32, total_frames // max_workers))
  - max_workers: execution_threads global (default: CPU count or 4)
  - Futures collected: future.result() waits for completion
  - Progress bar: tqdm tracks overall progress

  - ONNX inference dominates (already C++ optimized)

    * Type safety, async programming
    * Single binary distribution (Tauri)
    * Better cross-platform virtual camera support
    * 10-20% latency reduction via better threading

Keep (Pre-built Crates Available):

  - GUI: CustomTkinter -> Tauri + React/Vue
  - Core: Python threading -> tokio async runtime

  - Platform code: Scattered -> Organized FFI modules
